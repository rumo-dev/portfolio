Adaptationで露出調整済みのHDRカラーを最終的にディスプレイへ表示するため、AtmosFioreでは **Tone Mapping** を実装しています。ACES・Reinhard・AgX・GT（Uchimura）など、実に34種類のアルゴリズムを実行時に切り替えられるのが最大の特徴です。

---

## 📐 Tone Mappingの基本概念

HDRレンダリングでは輝度値が1.0を超える（真っ白より明るい）ピクセルが多数存在します。これをそのままSDRディスプレイ（0.0〜1.0）に出力すると白飛びが発生するため、非線形な圧縮関数（トーンカーブ）を通して滑らかにクランプする必要があります。

### 処理フロー

1. **Exposure適用済みHDRカラーの入力**
2. **選択したアルゴリズムでトーンマッピング**
3. **ガンマ補正**（`pow(color, 1.0 / gamma)`）
4. **SDRバッファ（R8G8B8A8_UNORM）への出力**

## 💻 C++実装

```cpp
class ToneMapping
{
public:
	struct alignas(16) ToneMappingConfig {
		int mapping_type = 0;
		float exposure = 1.0f;
		float gamma = 2.2f;
		float m_param = 0.22f;

		float max_white = 10.0f;
		float shoulder = 0.22f;
		float linear_strength = 0.30f;
		float linear_angle = 0.10f;

		float toe_strength = 0.20f;
		int is_enabled = 1;
		float padding[2]{};
	};

	enum class ToneMappingType {
		ACES = 0, Reinhard, Unreal, Neutral, Linear, Hable, AgX, GT,
		Drago, Exponential, Logarithmic, Ward, Lottes, Hejl,
		RomBinDaHouse, ReinhardExtended, FilmicSimple, ACESApprox,
		PBRNeutral, Sigmoid, Piecewise, Cineon, Exposure, GammaOnly,
		PQApprox, HLGApprox, OpenDRTLike, CameraResponse, Uchimura,
		ClampOnly, WhitePreservingLuma, FilmicALU, AgXPunchy, CustomCurve
	};

	ToneMapping(ID3D11Device* device, uint32_t width, uint32_t height);

	// 露出調整済みのHDRカラーをトーンマッピング・ガンマ補正する
	void make(ID3D11DeviceContext* immediate_context, ID3D11ShaderResourceView* adjusted_hdr_srv);

	// 最終的なSDR描画結果を取得 (Format: R8G8B8A8_UNORM)
	ID3D11ShaderResourceView* get_color_map() const { return tone_mapping_target->GetColorMap(); }

	ToneMappingType mapping_type = ToneMappingType::GT;
	float exposure = 1.0f;
	float gamma = 2.2f;
	float gt_param = 0.22f;
	float max_white = 10.0f;
	float shoulder = 0.22f;
	float linear_strength = 0.30f;
	float linear_angle = 0.10f;
	float toe_strength = 0.20f;
	int is_enabled = true;

private:
	std::unique_ptr<FullscreenQuad> bit_block_transfer;
	std::unique_ptr<Framebuffer>     tone_mapping_target;
	Microsoft::WRL::ComPtr<ID3D11PixelShader> tone_mapping_ps;
	Microsoft::WRL::ComPtr<ID3D11Buffer> cb_config;
};
```

## 💻 HLSL実装

### メインシェーダー（ディスパッチャ）

```hlsl
// tonemapping_ps.hlsl
#include "tonemapping_algorithms.hlsli"

float4 main(VS_OUT pin) : SV_TARGET
{
    float4 hdr_color = adjusted_hdr_texture.Sample(sampler_states[POINT_CLAMP], pin.texcoord);
    if (!is_enabled)
        return hdr_color;

    float3 color = hdr_color.rgb;
    color = ApplyToneMapping(color);

    // ガンマ補正
    color = pow(color, 1.0 / gamma);

    return float4(color, hdr_color.a);
}
```

### アルゴリズム例（`tonemapping_algorithms.hlsli`）

```hlsl
cbuffer ToneMappingConfig : register(b0)
{
    // 0:ACES 1:Reinhard 2:Unreal 3:Neutral 4:Linear 5:Hable 6:AgX 7:GT(Uchimura-like)
    // 8:Drago 9:Exponential 10:Logarithmic 11:Ward 12:Lottes 13:Hejl
    // 14:RomBinDaHouse 15:ReinhardExtended 16:FilmicSimple 17:ACESApprox
    // 18:PBRNeutral 19:Sigmoid 20:Piecewise 21:Cineon 22:Exposure 23:GammaOnly
    // 24:PQApprox 25:HLGApprox 26:OpenDRTLike 27:CameraResponse 28:Uchimura
    // 29:ClampOnly 30:WhitePreservingLuma 31:FilmicALU 32:AgXPunchy 33:CustomCurve
    int mapping_type;
    float exposure;
    float gamma;
    float m_param;
    float max_white;
    float shoulder;
    float linear_strength;
    float linear_angle;
    float toe_strength;
    bool is_enabled;
};

float GetLuma(float3 c)
{
    return dot(c, float3(0.2126, 0.7152, 0.0722));
}

// --- ACES Filmic（Narkowicz近似）---
float3 ToneMap_ACES(float3 x)
{
    const float a = 2.51;
    const float b = 0.03;
    const float c = 2.43;
    const float d = 0.59;
    const float e = 0.14;
    return saturate((x * (a * x + b)) / (x * (c * x + d) + e));
}

// --- Reinhard ---
float3 ToneMap_Reinhard(float3 x)
{
    return x / (1.0 + x);
}

// --- Reinhard Extended（白色点指定）---
float3 ToneMap_ReinhardExtended(float3 x)
{
    return (x * (1.0 + x / (max_white * max_white))) / (1.0 + x);
}
```

`ApplyToneMapping()` 内部では `mapping_type` に応じて上記のような34個の関数を分岐呼び出しし、GT/Uchimuraでは `shoulder` / `max_white`、CustomCurveでは `shoulder` / `linear_strength` / `linear_angle` / `toe_strength` など、アルゴリズムごとに異なるパラメータセットを参照します。

## 🎨 処理の詳細

### 1. アルゴリズムの選択指針

- **ACES / ACESApprox**：映画的な階調再現。ハイライトのロールオフが自然
- **AgX / AgXPunchy**：Blender発の彩度保持型トーンマッピング。彩度崩壊が少ない
- **GT（Uchimura）**：ゲーム向けに設計された、シャドウ・ハイライト双方を精密制御できる汎用カーブ
- **Reinhard系**：実装が単純で軽量だが、高輝度域での彩度低下が目立ちやすい

### 2. パラメータの相互依存

```hlsl
// GT / Uchimura, ReinhardExtended, Drago/Ward/Logarithmic で
// max_white や shoulder の意味が変化する
```

同じ `max_white` パラメータでも、アルゴリズムによって「白飛びの基準輝度」であったり「HDR白色点」であったりと意味が異なるため、C++側のGUIでは選択中のアルゴリズムに応じて表示するスライダーを動的に切り替えています。

### 3. ガンマ補正の分離

トーンマッピング関数自体は線形空間での圧縮のみを担当し、最終行の `pow(color, 1.0 / gamma)` でsRGB相当のガンマ補正を独立して適用しています。

## 🔧 実装のポイント

### 1. 34アルゴリズムの一元管理

すべてのアルゴリズムを1つの `.hlsli` ファイルに集約し、`mapping_type` の整数値で分岐することで、シェーダーの再コンパイルなしにC++側から動的に切り替え可能にしています。

### 2. パイプライン最終段としての配置

Tone Mappingは必ずAdaptation（自動露出）の後、パイプラインの最終段に配置されます。これより手前のすべてのエフェクトは線形HDR空間で処理され、Tone Mappingで初めてディスプレイ向けのSDR値に変換されます。

  デバッグGUIでは `ImGui::Combo`
  で34種類のアルゴリズム名をリスト表示し、選択中のアルゴリズムに応じて関連パラメータ（GT

  MappingをAdaptationより前段に配置すると、自動露出が正しく機能しません（露出調整はトーンマッピング前の線形HDR値に対して行う必要があるため）。
</Aside>
